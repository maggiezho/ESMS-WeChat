# 基于微信小程序的快递驿站管理系统

## 一、项目概述

### 1.1 项目名称

快递驿站管理系统

### 1.2 项目简介

快递驿站管理系统是一款基于微信小程序云开发技术的快递末端管理应用。系统面向普通用户和驿站管理员两类角色，提供快递查询、扫码取件、扫码入库、快递管理、数据统计等核心功能。普通用户可通过手机号查询本人名下的待取快递，通过扫码完成取件操作；管理员可通过扫码或手动录入方式完成快递入库，对快递信息进行编辑、删除等管理操作，并查看今日入库和今日取件统计数据。系统采用微信小程序原生框架开发，使用云数据库进行数据持久化存储，云函数实现服务端逻辑，无需自行搭建服务器即可快速部署使用。

### 1.3 开发环境与技术栈

| 项目 | 说明 |
|------|------|
| 开发工具 | 微信开发者工具（最新稳定版） |
| 前端框架 | 微信小程序原生框架（WXML + WXSS + JavaScript） |
| 后端服务 | 微信小程序云开发（CloudBase） |
| 数据库 | 云数据库（文档型NoSQL数据库） |
| 云函数运行环境 | Node.js |
| 部署方式 | 微信小程序云开发一键部署 |


## 二、功能说明

### 2.1 用户端功能模块

#### 2.1.1 手机号查询快递

用户进入首页后，在输入框中输入收件人的11位手机号，点击“查询”按钮，系统自动检索数据库中该手机号对应的所有快递记录。查询结果按“待取”和“历史”两个标签页分别展示，标签页上方显示各类别的数量统计。

#### 2.1.2 待取列表展示

“待取”标签页中展示该手机号下所有状态为“待取”的快递，每条记录以卡片形式呈现，包含取件码、货架位置、入库时间等关键信息。卡片底部设有“扫码取件”按钮，方便用户直接操作。

#### 2.1.3 历史记录查看

“历史”标签页中展示该手机号下所有状态为“已取”的快递，包含取件码、货架位置和取件时间。历史记录仅供查看，不可进行取件操作。

#### 2.1.4 扫码取件

用户在待取列表中点击某快递卡片的“扫码取件”按钮，系统调起微信扫码功能，扫描驿站提供的取件二维码。系统将扫描结果中的取件码与当前快递进行比对，匹配成功后弹出确认对话框，用户确认后快递状态更新为“已取”并记录取件时间。

#### 2.1.5 空状态提示

当用户未查询或查询后无结果时，页面展示相应的空状态引导界面，包括未查询时的搜索引导、无待取快递提示和无历史记录提示，避免白屏，提升用户体验。

### 2.2 管理端功能模块

#### 2.2.1 管理员身份验证

点击底部“管理”标签页时，系统自动弹出密码输入弹窗，管理员需输入正确密码（默认密码：123456）方可进入管理页面。密码校验通过云函数在服务端完成，保障管理功能的安全性。

#### 2.2.2 数据统计看板

管理页面顶部展示“今日入库”和“今日取件”两个统计卡片。今日入库统计createTime为当天的快递数量，今日取件统计takeTime为当天的快递数量，帮助管理员快速了解当日业务概况。

#### 2.2.3 扫码入库（智能解析）

管理员点击“扫码入库”按钮后调起微信扫码功能，扫描快递二维码。二维码采用“取件码 | 手机号”格式（如“53-4-2001 | 13812345678”），系统自动解析出取件码和收件人手机号，并根据取件码格式（如“53-4-2001”）自动识别出货架位置（“53-4”），随后跳转至添加页面并自动填充上述信息，管理员确认后即可完成入库。

#### 2.2.4 手动添加快递

管理员点击“添加快递”按钮，跳转至添加页面，手动输入取件码和收件人手机号。取件码必须符合“货架号-层数(1-5)-四位数字”的格式规范，系统自动校验格式并识别货架位置，提交后快递入库。

#### 2.2.5 快递列表展示与搜索

管理页面提供“待取件”和“全部快递”两个标签页，分别展示不同状态的快递。搜索栏支持按手机号或取件码进行关键词筛选，帮助管理员快速定位特定快递。

#### 2.2.6 快递编辑与删除

在待取快递列表中，每个卡片下方提供“编辑”和“删除”按钮。点击“编辑”跳转至编辑页面，可修改取件码和收件人手机号；点击“删除”弹出二次确认对话框，确认后从数据库移除该记录。

#### 2.2.7 短信提醒（模拟）

管理员点击快递卡片上的“提醒”按钮，系统弹出模拟提示框，展示即将发送通知的收件人手机号。当前为模拟实现，为后续接入真实短信或订阅消息预留接口。

#### 2.2.8 管理员扫码取件

管理员可点击“扫码取件”按钮，扫描用户提供的取件二维码，系统根据取件码查询对应的待取快递，确认后完成取件操作。该功能便于管理员协助不熟悉操作的用户取件。

### 2.3 数据库设计

系统使用一个名为 `parcels` 的云数据库集合，各字段说明如下：

| 字段名 | 数据类型 | 必填 | 说明 |
|--------|----------|------|------|
| _id | String | 是 | 系统自动生成的唯一标识符 |
| _openid | String | 是 | 用户openid，系统自动添加 |
| trackingCode | String | 是 | 取件码，格式：货架号-层数-四位码，如“53-4-2001” |
| recipientPhone | String | 是 | 收件人手机号，11位数字 |
| shelfNo | String | 是 | 货架位置，由取件码自动解析，如“53-4” |
| status | Number | 是 | 快递状态，0=待取，1=已取 |
| createTime | Date | 是 | 入库时间，自动生成 |
| takeTime | Date | 否 | 取件时间，取件时记录 |
| takenByAdmin | Boolean | 否 | 是否由管理员协助取件 |


## 三、使用指南

### 3.1 用户端使用流程

**步骤一：进入首页** — 启动小程序后默认进入“快递查询”首页，页面顶部展示品牌标识和欢迎语，中部为手机号输入区域。

**步骤二：输入手机号查询** — 在输入框中输入收件人的11位手机号，点击“查询”按钮。若输入格式不正确（非11位数字），系统会给出提示信息。

**步骤三：查看待取快递** — 查询成功后，默认展示“待取快递”标签页，列表中显示所有待取快递的取件码、货架位置和入库时间。页面顶部显示待取数量。

**步骤四：扫码取件** — 在驿站找到对应货架上的包裹后，点击该卡片底部的“扫码取件”按钮，扫描包裹上的取件二维码。系统验证取件码匹配后，点击确认即可完成取件。

**步骤五：查看历史记录** — 点击“历史记录”标签页，可查看已取件的快递记录，包含取件时间信息。

### 3.2 管理端使用流程

**步骤一：进入管理页** — 点击底部TabBar的“管理”标签页，系统弹出密码输入弹窗，输入管理密码（默认123456）。

**步骤二：快递入库** — 有两种方式：（1）点击“扫码入库”，扫描快递二维码后自动填充信息，确认后完成入库；（2）点击“添加快递”，手动输入取件码和手机号，提交完成入库。

**步骤三：查看与管理快递** — 管理页面展示快递列表，可通过“待取件/全部快递”标签页切换视图，也可通过搜索栏按手机号或取件码快速筛选。

**步骤四：编辑或删除快递** — 在待取快递卡片中点击“编辑”可修改快递信息，点击“删除”可移除快递记录。

**步骤五：扫码协助取件** — 点击“扫码取件”扫描用户二维码，可帮助用户完成取件操作。

### 3.3 部署说明

#### 3.3.1 创建云开发环境

1. 在微信开发者工具中点击“云开发”按钮，创建新的云开发环境
2. 记录环境ID，在 `app.js` 中替换 `env: '你的环境ID'`

#### 3.3.2 创建数据库集合

在云开发控制台中创建名为 `parcels` 的集合，权限设置为“所有用户可读，仅创建者可写”，或根据实际需求配置。

#### 3.3.3 部署云函数

在 `cloudfunctions/checkAdmin` 目录右键选择“上传并部署”，将管理员验证云函数部署至云端。可在 `index.js` 中修改 `ADMIN_PASSWORD` 变量值以更换管理密码。

#### 3.3.4 配置项目

1. 检查 `app.json` 中的页面路径配置是否正确
2. 准备TabBar所需的图片资源（query.png、query-active.png、admin.png、admin-active.png）放置在 `images` 目录下


## 四、核心代码辅助说明

### 4.1 取件码格式校验与货架位置自动解析

系统使用正则表达式对取件码进行格式校验，确保入库数据规范性。取件码合法格式为“货架号-层数(1-5)-四位数字”（如“53-4-2001”），校验成功时自动提取出货架位置。

**涉及文件：** `pages/add/add.js` 中的 `parseAndValidateTrackingCode(trackingCode)` 函数

该函数使用 `/^(\d+)-([1-5])-(\d{4})$/` 正则表达式进行匹配，匹配成功后从取件码中截取“货架号-层数”作为货架位置，并返回校验状态及对应的提示信息。校验失败时会根据错误类型（如层数超出1-5范围、四位码不足4位、格式不完整等）返回不同的错误描述，便于前端展示友好的用户提示。

### 4.2 管理员扫码入库智能解析

管理员通过扫码入库时，系统需解析二维码内容中的取件码和手机号，并自动识别货架位置。

**涉及文件：** `pages/admin/admin.js` 中的 `adminScanAdd()` 函数和 `parseShelfNoFromTrackingCode(trackingCode)` 函数

扫码成功后，系统首先将二维码内容按竖线分隔符拆解为取件码和手机号两部分，然后调用解析函数从取件码中提取货架位置。解析失败时会引导管理员进入手动添加页面，并自动填充已提取的部分信息，避免完全重复录入。

### 4.3 用户端并行查询优化

用户查询时需同时加载待取列表和历史记录，系统采用并行查询策略提升响应速度。

**涉及文件：** `pages/index/index.js` 中的 `loadAllData(phone)` 函数

该函数使用 `Promise.all()` 方法同时发起两个数据库查询请求，分别查询 `status=0`（待取）和 `status=1`（已取）的记录，待两个请求全部完成后统一处理返回数据并更新页面。相比串行查询，显著减少了用户等待时间。

### 4.4 管理员密码校验云函数

管理员密码校验在服务端完成，前端仅传递用户输入的密码，不存储或比对密码。

**涉及文件：** `cloudfunctions/checkAdmin/index.js`

云函数接收前端传递的 `password` 参数，与服务端预设的 `ADMIN_PASSWORD` 常量进行比对，返回 `{ success: true }` 或 `{ success: false }` 结果。由于校验逻辑运行在云端，前端无法直接获取密码常量，保障了管理功能的安全性。


## 五、注意事项

1. 首次使用前需在 `app.js` 中配置正确的云开发环境ID
2. 管理密码默认为 `123456`，可在 `cloudfunctions/checkAdmin/index.js` 中修改 `ADMIN_PASSWORD` 变量值
3. 二维码格式需遵循“取件码 | 手机号”的格式约定，取件码格式为“货架号-层数-四位码”
4. 当前版本短信提醒为模拟功能，未对接真实短信服务商
5. 建议在微信开发者工具中使用真机调试，以测试扫码功能的完整流程
6. 确保云数据库权限配置正确，避免数据读写失败
